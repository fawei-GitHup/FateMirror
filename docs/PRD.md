# 命镜 FateMirror — 产品需求文档 (PRD)

**版本：** v3.1 (认知诊断增强版)
**日期：** 2026-04-05
**定位：** AI驱动的人生模式识别与认知突破引擎，游戏化成长日志
**核心主张：** 不只告诉你重复了什么，更要照见你为什么重复。

---

## 一、产品愿景

### 1.1 一句话描述
FateMirror 是一款将个人日记转化为可视化"命运树"的 AI 成长工具，通过识别用户反复踩坑的行为模式，在即将重蹈覆辙时用用户**自己的历史原话**进行预警，帮助用户真正打破人生循环。

### 1.2 为什么这个产品应该存在
市面上所有 AI 日记产品（Rosebud Journal、Reflectr、Jour）都在做同一件事：**帮你"看见"自己的模式。**

但"看见"和"改变"之间有一条巨大的鸿沟。

FateMirror 要做的不是告诉你"你有焦虑倾向" — 你早就知道了。而是在你**第三次因为同样的原因准备掏空钱包**的那一刻，把你两年前写的那篇《又一次》原文拉出来摆在你面前。

没有任何通用 AI 能做到这一点，因为它们没有你的历史记忆。

### 1.3 核心差异化

| 维度 | Rosebud Journal | FateMirror |
|------|----------------|------------|
| 定位 | AI 日记工具 | 人生成长 RPG |
| 核心能力 | 情绪分析 + 周报 | **循环检测 + 历史原话打脸** |
| 知识体系 | 纯西方心理学 | 东西方融合（易经/斯多葛/CBT） |
| 可视化 | 图表/周报 | **命运树（人生分叉地图）** |
| 游戏化 | 几乎没有 | 境界系统/章节制/心魔挑战 |
| AI 人格 | 中性引导者 | 老墨（毒舌智者 IP） |
| 文化 | 英文圈 | 英文先发，中文原生设计 |

## 二、目标用户

### 2.1 首发用户画像（海外英文圈）

**主画像：The Stuck Achiever（困在原地的行动者）**
- 25-40 岁，有一定职业基础但感到停滞
- 读过《原子习惯》《被讨厌的勇气》等自我提升书籍
- 能识别自己的问题模式（"我知道我不该这样"）但无法打破
- 对东方哲学有好奇心（冥想/禅修在欧美已主流化）
- 月可支配收入 $3,000+，对 $9.9/月不敏感
- 活跃于 Reddit r/selfimprovement、r/productivity、r/stoicism

**次画像：The Digital Journaler（数字日记用户）**
- 已经有日记习惯（Day One / Notion / 纸质笔记本）
- 觉得"写了很多但没什么用"，渴望 AI 的洞察力
- 可能已经试过 Rosebud 但觉得"AI回复太鸡汤"

### 2.2 Phase 2 用户画像（中文圈）
- 海外华人留学生/工作者（无需备案即可访问）
- 一二线城市 25-35 岁职场人
- 内卷焦虑但有自我觉察意识
- 对"命理""国学"天然亲近但对"封建迷信"保持距离

### 2.3 非目标用户（明确排除）
- 寻求专业心理治疗的患者 → 产品不是医疗工具，需要免责声明
- 期望占卜/算命的用户 → 产品基于心理学+哲学，不做神秘主义
- 没有反思习惯且不愿意写字的用户 → 产品核心是写作

## 四、认知诊断框架 — 产品的理论内核

> 上一版PRD只解决了"你重复了什么"，这一版要回答**"你为什么重复"**。
> 根因是三层结构：**思维模式 → 行为模式 → 习惯固化**。
> 改变人生 = 先照见这三层，然后逐层修正。

### 4.1 三层认知金字塔模型 (Cognition Pyramid)

这是命镜的理论内核。所有功能都围绕这个模型展开。

```
                    ▲
                   / \
                  / 习 \          第3层：习惯固化层 (Habit Layer)
                 / 惯层 \         思维+行为反复执行后凝固的自动化程序
                /────────\        表现：不需要思考就做了，做完才后悔
               /          \
              /  行为模式层  \     第2层：行为模式层 (Behavior Layer)
             /              \     面对特定场景的默认反应方式
            /────────────────\    表现：知道不该这样但身体比脑子快
           /                  \
          /    思维模式层       \   第1层：思维模式层 (Thinking Layer)
         /                      \  看问题的视角、宽度、深度
        /________________________\  表现：只看到点，看不到线、面、系统
```

### 4.2 思维模式层 — 思维视野诊断

**核心洞察：** 大部分人困在原地，不是因为不努力，而是因为**看问题的维度不够**。

#### 4.2.1 思维视野四级模型

| 级别 | 名称 | 特征 | 典型表现 | 产品中的诊断标签 |
|------|------|------|---------|----------------|
| L1 | **点状思维** (Point) | 只看到眼前一个事件 | "又吵架了" → 只想着怎么哄好 | `thinking:point` |
| L2 | **线性思维** (Linear) | 能看到因果链但只有一条 | "因为钱不够 → 所以她不开心" | `thinking:linear` |
| L3 | **面状思维** (Systemic) | 能看到多条因果链交织成网 | "钱的问题 + 沟通方式 + 她的安全感需求 + 我的自我价值感 都在互相影响" | `thinking:systemic` |
| L4 | **系统思维** (Meta) | 能看到系统的结构并找到杠杆点 | "根本不是钱的问题，是我用'给钱'来回避真正的沟通，这个模式来自原生家庭" | `thinking:meta` |

#### 4.2.2 AI 如何诊断思维级别

**不是问卷测评，是从日记文本中自然提取：**

```
诊断信号：

L1 点状思维的语言特征：
  - 频繁使用"就是因为..."（单因归因）
  - 大量"他/她让我..."（外部归因为主）
  - 很少出现"另一方面"、"同时"等多维连接词
  - 日记中只描述事件，不分析原因

L2 线性思维的语言特征：
  - 能说出"因为A所以B"
  - 但因果链只有一条，缺乏分支
  - 解决方案也是线性的："只要我赚到钱就好了"

L3 面状思维的语言特征：
  - 出现"一方面...另一方面..."
  - 能识别多个变量之间的相互影响
  - 但可能陷入分析瘫痪："想太多反而不知道怎么做"

L4 系统思维的语言特征：
  - 能识别深层结构（"这个模式来自..."）
  - 能找到杠杆点（"如果我改变X，整个局面会变"）
  - 能区分症状和病因
```

#### 4.2.3 老墨如何引导思维升级

**不是教用户"你应该系统思考"，而是用提问把用户从L1拉到L2、L2拉到L3：**

```
场景：用户写"又因为钱和女朋友吵架了"

如果AI诊断用户当前处于 L1（点状思维）：
  老墨：你说"因为钱"。但上次你赚到了一笔钱，
       你们还是吵了。所以——真的是钱的问题吗？
       ↑ 这个提问引导用户从L1→L2（发现单因归因不成立）

如果用户已经到 L2（线性思维）：
  老墨：你说"钱不够→她不开心→我愧疚→我更拼命赚→更没时间陪她→她更不开心"。
       这条链你看得很清楚。但你有没有想过——
       她要的真的是钱，还是"被看见"？
       你拼命赚钱这个行为本身，是不是在回避什么？
       ↑ 引导从L2→L3（发现这不是一条线，是一个交织的网）

如果用户已经到 L3（面状思维）：
  老墨：你分析得很透彻——钱、沟通、安全感、自我价值，
       四条线搅在一起。但你还是不知道从哪下手对吧？
       让我问你一个问题：这四条线里，
       哪一条不是"别人让你这样的"，而是"你自己选择这样的"？
       ↑ 引导从L3→L4（找到自己可以改变的杠杆点）
```

### 4.3 行为模式层 — 场景化行为诊断

**核心洞察：** 人在特定场景下的默认反应是可预测的。"知道不该这样但就是控制不住"，因为行为模式比思考快。

#### 4.3.1 六大行为模式原型

从心理学和行为经济学提炼，覆盖绝大多数"重复踩坑"的行为：

| 原型 | 英文 | 特征描述 | 典型场景 | 产品标签 |
|------|------|---------|---------|---------|
| **过度补偿者** | Over-Compensator | 用A领域的过度付出来弥补B领域的不安全感 | 赚不到钱→拼命对人好；感情不安→拼命花钱 | `behavior:over-compensate` |
| **回避者** | Avoider | 面对真正的问题时选择绕路 | 该谈钱的时候谈感情，该直面冲突的时候冷战 | `behavior:avoid` |
| **讨好者** | People-Pleaser | 牺牲自己边界来维持关系 | 答应做不到的事，不敢说"不" | `behavior:please` |
| **控制者** | Controller | 通过控制外部变量来缓解内部焦虑 | 过度计划、微管理、不愿授权 | `behavior:control` |
| **证明者** | Prover | 通过成就来验证自我价值 | 停不下来、害怕被看作不够好 | `behavior:prove` |
| **受害者** | Victim-Loop | 将失败归因于外部、命运、他人 | "又是这样"、"为什么倒霉的总是我" | `behavior:victim` |

**关键：一个人通常有2-3个主导行为模式叠加。**

比如你描述的"感情中总是主动付出但因钱失败" → 可能是 **过度补偿者 + 讨好者** 叠加。

#### 4.3.2 AI 如何标注行为模式

```
检测逻辑（从日记文本中提取）：

过度补偿者信号：
  - 日记中出现"至少我可以..."、"我能做的就是..."
  - A领域受挫后，B领域行为突然加强
  - 付出与回报严重不对等但仍持续

回避者信号：
  - "算了"、"不想再说了"、"等以后再..."
  - 日记提到冲突但没有任何面对行动
  - 同一个问题被提到3次以上但从未直接处理

讨好者信号：
  - "我答应了..."、"不好意思拒绝"
  - 日记中大量"应该"（for others）
  - 很少出现"我想要"、"我需要"

每篇日记AI输出增加字段：
  behavior_patterns: ['over-compensate', 'please']
  behavior_evidence: "用户提到'把工资全给她让她开心'——典型的过度补偿:用金钱付出弥补关系中的不安全感"
```

### 4.4 习惯固化层 — 习惯回路诊断

**核心洞察：** 思维模式和行为模式反复执行后会变成"自动驾驶"。你不是"选择"这样做的，你是"习惯性"这样做的。

#### 4.4.1 习惯回路模型（基于《原子习惯》简化）

```
┌─────────┐     ┌─────────┐     ┌─────────┐     ┌─────────┐
│  触发    │ ──→ │  渴求    │ ──→ │  反应    │ ──→ │  奖赏    │
│  Cue     │     │  Craving │     │ Response │     │  Reward  │
└─────────┘     └─────────┘     └─────────┘     └────┬────┘
                                                      │
                              ┌────────────────────────┘
                              │ 强化循环（每次奖赏都在加固这个回路）
                              ▼
                        下次触发时更自动
```

**以你的场景为例：**
```
触发：女朋友表达不满/失望
渴求：消除愧疚感、避免冲突升级
反应：立刻用金钱补偿（给钱/买东西/答应旅行）
奖赏：短期冲突平息，对方暂时高兴

→ 回路强化：下次遇到同样触发，金钱补偿的反应更快更自动
→ 副作用：长期经济压力增大，对方期望阈值抬高，问题根源从未被触碰
```

#### 4.4.2 AI 如何识别并呈现习惯回路

当同一个 behavior_pattern 被触发 ≥ 3 次时，AI 自动生成完整的习惯回路分析：

```
老墨的习惯回路报告：

📊 你有一个运行了至少8个月的自动程序：

  触发器：当你感到女朋友对你失望时
  ↓
  你的自动反应：用钱来"修补"
  ↓
  短期奖赏：冲突暂时平息
  ↓
  隐性代价：经济压力↑ + 对方期望↑ + 根本问题未触碰

  这个回路你跑了至少 3 次（2024-03, 2024-10, 2025-01）。

  你自己在 2024-10-15 就发现了这个回路：
  > "每次用钱解决问题，短期她高兴了，但我内心更空了"

  但发现回路不等于打破回路。
  打破它的方法不是"下次不给钱"——那只是压制反应。
  而是改变触发器到反应之间的那一步：
  当你感到她失望时，你的第一个念头是什么？
  能不能把那个念头说出来，而不是掏钱包？
```

### 4.6 认知标签体系（完整）

将以上理论转化为产品中的结构化标签系统：

```
标签类别 1：情绪标签 (emotions) — 已有
  anxiety | guilt | anger | helplessness | joy | pride | calm | ...

标签类别 2：主题标签 (themes) — 已有
  money | relationship | career | family | self-worth | health | ...

标签类别 3：思维模式标签 (thinking_patterns) — 新增
  thinking:point           # 点状思维（单因单果）
  thinking:linear          # 线性思维（单链因果）
  thinking:systemic        # 面状思维（多维分析）
  thinking:meta            # 系统思维（结构性洞察）

  -- 常见认知扭曲子标签（CBT）
  distortion:catastrophize # 灾难化（"完了，一切都毁了"）
  distortion:black-white   # 非黑即白（"要么全好要么全坏"）
  distortion:overgeneralize # 过度概括（"我总是..."）
  distortion:mind-reading  # 读心术（"她肯定觉得我没用"）
  distortion:should        # 应该化（"我应该做到..."）
  distortion:emotional-reasoning # 情绪推理（"我觉得是这样→所以就是这样"）
  distortion:personalize   # 个人化（"都是我的错"）
  distortion:filter        # 心理过滤（只看到负面）

标签类别 4：行为模式标签 (behavior_patterns) — 新增
  behavior:over-compensate # 过度补偿
  behavior:avoid           # 回避
  behavior:please          # 讨好
  behavior:control         # 控制
  behavior:prove           # 证明
  behavior:victim          # 受害者循环

标签类别 5：习惯回路标签 (habit_loops) — 新增
  每个被识别的回路生成一个结构化记录：
  {
    id:          UUID
    cue:         string     // 触发器描述
    craving:     string     // 渴求描述
    response:    string     // 反应描述
    reward:      string     // 奖赏描述
    hidden_cost: string     // 隐性代价
    trigger_count: int      // 运行次数
    status:      'active' | 'observed' | 'breaking' | 'broken'
    related_behavior: string // 关联的行为模式
    related_thinking: string // 关联的思维模式
  }
```

### 4.7 认知升级路径 — 从诊断到改变

**诊断不是目的，改变才是。** 命镜的认知升级分四步：

```
Step 1：照见 (See)
  "你有一个'过度补偿'的行为模式"
  → 用户反应：震惊/否认/好奇
  → 产品行为：展示证据（3篇历史日记佐证）
  → 老墨风格："不是我说的，你自己写的。看看这三篇日记。"

Step 2：理解 (Understand)
  "这个模式可能来自于你的思维方式——你把'给钱=表达爱'当成了唯一因果链"
  → 产品行为：展示习惯回路图 + 思维维度分析
  → 老墨风格："你不是坏人，你是程序跑偏了。我们来看看这个程序的源代码。"

Step 3：中断 (Interrupt)
  下次触发时，实时干预
  → 产品行为：当新日记匹配到已知回路 → 弹出预警 + 历史原话
  → 老墨风格："等一下。上次你跑了这个程序，结果你自己说了什么？"

Step 4：替换 (Replace)
  不能只说"别这样做"，必须给出"那该怎么做"
  → 产品行为：基于思维升级框架，生成替代行动方案
  → 老墨风格：
    "你习惯掏钱包。这次试试掏心窝子——
     跟她说：'我知道你失望了，我也很难受。我们能不能坐下来，
     不谈钱，先谈谈你最需要的到底是什么？'
     就这一句话。回来告诉我她怎么说的。"
```

### 4.8 "穷人思维 vs 富人思维"框架集成

你特别提到了思维方式的贫富差异。这不是"富人更聪明"，而是**思维的时间维度和系统维度不同**：

| 维度 | 匮乏思维 (Scarcity) | 富足思维 (Abundance) | AI诊断信号 |
|------|---------------------|---------------------|-----------|
| **时间维度** | 只看当下（这个月怎么过） | 看长期（3年后会怎样） | 日记中时间词汇分析：大量"今天""这周" vs "明年""未来" |
| **因果宽度** | 单因归因（"因为没钱"） | 多因分析（"资源配置+能力结构+机会成本"） | 归因分析中变量数量 |
| **风险态度** | 恐惧失去 > 渴望获得 | 计算风险，接受合理损失 | 决策描述中"怕""不敢" vs "值得试""最差也就" |
| **资源观** | 零和博弈（我多他就少） | 正和博弈（共同做大蛋糕） | 人际关系描述中"吃亏""占便宜" vs "合作""共赢" |
| **自我投资** | 视学习为成本 | 视学习为投资 | 花钱决策分析：消费支出 vs 能力建设 |
| **行动模式** | 等"准备好了"再行动 | 边做边学，快速迭代 | 日记中"等我...再..."的频率 |

**AI在日记分析中标注这些维度，但不直接贴"穷人思维"标签（太刺耳），而是用老墨的方式点出来：**

```
老墨：你写了"等我攒够了钱再去学那个课程"。
     我帮你算一笔账：
     这个课程可能帮你多赚的钱 ÷ 学费 = 回报率。
     你在金融行业，这个概念你应该比我懂才对。

     为什么轮到自己身上，你突然变成了"先攒钱再投资"？
     这是你在投资领域也会犯的同一个错误吗？
```

## 五、核心功能详细设计

### 5.1 F2 - AI 引导式日记（含认知诊断增强）

**交互流程：**

```
┌─────────────────────────────────────────────┐
│  用户点击"新建日记" → 选择模式                 │
│                                             │
│  [🗣️ 老墨引导]     [✍️ 自由书写]              │
│                                             │
│  选择"老墨引导"后：                            │
├─────────────────────────────────────────────┤
│                                             │
│  老墨：今天什么事让你心里最不平静？              │
│                                             │
│  用户：和女朋友又因为钱吵架了...                │
│                                             │
│  老墨：嗯。吵架的导火索是什么具体的事？          │
│       不用美化，就说真话。                      │
│                                             │
│  用户：她说我答应的旅行又泡汤了因为钱不够...      │
│                                             │
│  老墨：你说"又"。这不是第一次了对吧？            │
│       上一次因为钱让她失望是什么时候？            │
│                                             │
│  ... (3-5轮后自动整理成结构化日记)              │
│                                             │
├─────────────────────────────────────────────┤
│  [💾 保存日记]  [🌳 查看命运树]  [📋 生成行动]  │
└─────────────────────────────────────────────┘
```

**AI 引导策略（7轮深度引导）：**
- 第1轮：开放倾倒 — "今天什么事让你心里最不平静？"
- 第2轮：聚焦事件 — "具体发生了什么？不美化，说真话。"
- 第3轮：追问感受 — "当时你心里什么感觉？身体有什么反应？"
- 第4轮：连接模式 — "这让你想起之前的哪次经历？"
- 第5轮：思维探照 — "你说'因为没钱'——除了钱，还有什么在影响这件事？"（引导从点→线→面）
- 第6轮：行为回溯 — "你当时的第一反应是什么？你有没有想过另一种反应？"（探测行为模式）
- 第7轮：提炼洞察 — "如果未来的你回头看今天，你希望自己做了什么不同的事？"

**结束时自动提取（六维标签）：**
- 情绪标签 emotions[]（焦虑/愧疚/愤怒/无力/...）
- 主题标签 themes[]（金钱/感情/工作/自我价值/...）
- 关键决策 decisions[]（"决定把工资全给她"）
- 用户洞察原话 insights[]（"每次用钱解决问题短期有效但长期更空"）
- **思维模式标签 thinking_patterns[]**（`thinking:linear` + `distortion:overgeneralize`）
- **行为模式标签 behavior_patterns[]**（`behavior:over-compensate` + `behavior:please`）

### 4.2 F5 - 循环预警（核心差异化功能）

**触发逻辑：**

```
当用户写下新日记时：
1. 提取当前日记的 themes[] 和 emotions[]
2. 查询历史日记中 themes 重叠度 > 60% 的条目
3. 如果匹配到 ≥ 2 篇历史日记，且时间跨度 > 30 天
   → 判定为"循环模式"
4. 从匹配的历史日记中提取 user_insights（用户反思原话）
5. 将历史原话注入 AI Prompt，让老墨用这些原话来干预
```

**AI 干预示例：**

```
老墨：等一下。我需要你停下来看一样东西。

你在 2024-10-15 写过这段话：
> "每次我用钱解决问题，短期她高兴了，
>  但我内心更空了，而且下次她的期望更高。"

今天你又准备做同样的事。
这已经是第三次了。

我不是说你不该给钱。
我是问 — 上次你说要"先谈清楚再决定怎么花"，
这次你谈了吗？
```

**循环预警的关键设计原则：**
1. **只用用户自己的话** — 不说教，不灌鸡汤，你自己说的
2. **指出次数** — "这是第三次了"比"你有这个倾向"震撼10倍
3. **不替用户做决定** — "我不是说你不该..."，保留用户自主权
4. **追问行动** — "上次你说要...这次你做了吗？"

### 4.3 F6 - 命运树

**数据模型：**

```
LifeNode {
  id:              UUID
  userId:          UUID
  title:           string       // "感情因钱第三次崩盘"
  date:            date         // 2024-10
  type:            enum         // milestone | choice | insight | crisis
  themes:          string[]     // ['money', 'relationship']
  emotions:        string[]     // ['guilt', 'helplessness']
  parentId:        UUID | null  // 连接到父节点
  journalIds:      UUID[]       // 关联的日记ID
  aiSummary:       string       // AI生成的节点摘要
  userReflection:  string | null // 用户自己写的反思
  status:          enum         // active | resolved | recurring
  isAutoGenerated: boolean      // true=AI自动生成, false=用户手动添加
}
```

**树的生长逻辑：**
1. 用户每写一篇日记，AI判断是否构成新的"人生节点"
2. 日常小事 → 不生成节点（避免树过于密集）
3. 重大决策/情绪事件/模式重复 → 自动生成节点
4. 用户可以手动添加/编辑/删除节点
5. 相似主题的节点之间自动画出"因果连线"（虚线）

**视觉规格（MVP - 2D）：**
- 暗色背景 + 发光节点 + 渐变连接线
- 节点颜色编码：crisis=红, insight=金, choice=蓝, milestone=绿
- 点击节点 → 展开侧面板：显示关联日记摘要 + AI分析
- 缩放/平移支持（大树需要导航）
- recurring（重复模式）节点之间用醒目的红色虚线连接

### 4.4 F8 - 境界系统

**等级定义：**

| 等级 | 名称 | 英文 | 解锁条件 | 解锁能力 |
|------|------|------|---------|---------|
| 0 | 沉睡者 | Sleeper | 注册 | 基础日记功能 |
| 1 | 萌芽者 | Seedling | 写第1篇日记 | 解锁命运树基础视图 |
| 2 | 观察者 | Observer | 累计7篇日记 | 解锁模式检测 |
| 3 | 映照者 | Mirror | 首次承认自己的循环模式 | 解锁历史原话回溯 |
| 4 | 破局者 | Breaker | 在循环预警时选择了不同行动并坚持7天 | 解锁深度AI分析 |
| 5 | 扎根者 | Rooted | 连续30天保持新行为模式 | 解锁章节总结 |
| 6 | 明镜 | Mirror Master | 累计打破3个循环模式 | 全部功能解锁 |

**升级条件核心原则：不是"写了多少篇"，而是"行为是否真正改变了"。**

AI 判定升级的逻辑：
- Lv.3 → Lv.4：用户在循环预警后，日记中出现了与以往不同的决策描述
- Lv.4 → Lv.5：连续30天内，同一主题的日记情绪标签从负面转为中性/正面

### 4.5 F9 - 章节自动生成

**生成逻辑：**
- 每当用户累计写满15篇日记，AI对这批日记做主题聚类
- 自动生成章节名称和摘要
- 章节划分也可由"重大事件"触发（如检测到人生转折点）

**章节结构：**
```

  dateRange:  [date, date] // 起止时间
  themes:     string[]     // 本章核心主题
  treeNodes:  UUID[]       // 本章包含的命运树节点
  aiNarrative: string      // AI生成的章节叙述（200字）
  status:     'open' | 'closed'
}
```

### 5.4 Prompt 结构（含认知诊断增强）

```
System Prompt = {
  1. 老墨人格定义（300 tokens）
  2. 当前用户认知画像（从数据库读取，400 tokens）
     ├── 基础信息（人生背景/价值观）

     ├── 行为模式TOP3 + 置信度

     ├── 如果行为=过度补偿→注入边界设定相关哲学
     └── 如果习惯回路活跃→注入回路中断策略
  4. 当前日记上下文（用户本次输入，变长）
  5. 历史匹配日记+认知标注（模式检测命中时注入，≤ 2000 tokens）
     ├── 匹配的历史日记摘要
     ├── 用户当时的思维/行为标注
     ├── 用户自己的反思原话
     └── 与当前的对比（"上次L1单因归因，这次有无变化？"）
  6. 指令约束（300 tokens）
     ├── 不做诊断/不替用户做决定/引用原话
     ├── 认知标签不直接展示给用户（用对话自然引出）
     ├── 思维升级引导通过提问而非说教
     └── 每次输出包含结构化标签JSON（隐藏，供系统存储）
}

总计控制在 5000-7000 tokens，留足输出空间
```

每次AI回复实际包含两部分：用户可见的对话 + 系统隐藏的结构化标注。

## 六、隐私与信任设计

### 6.1 信任建立四层漏斗

```
第1层：降低感知门槛
  → 产品定位是"growth journal"而非"therapy tool"
  → 用户心理：写日记 <<<< 看心理医生
  → 第一次使用不需要暴露任何私密信息

第2层：渐进式信任
  → 免费就能完整使用核心功能（限制次数）
  → AI第一次精准击中用户内心时，信任自然建立
  → 写了几篇后AI越来越"懂你" → 沉没成本锁定

第3层：透明承诺
  → "We never read your journals"
  → "We never sell your data"
  → "All data encrypted in transit and at rest"
  → 在设置页明确展示数据处理方式

第4层：用户控制权
  → 一键导出所有数据（JSON/Markdown）
  → 一键删除账户和所有数据（真删除，不是标记）
  → 每次AI调用前可预览发送的内容（可选开关）
```

### 6.2 技术隐私措施
- Supabase RLS（行级安全）：用户只能访问自己的数据
- API 调用：日记内容直传 AI 供应商，不在自己服务器存留明文（AI供应商有API数据不训练协议）
- 本地脱敏选项：可选的正则脱敏（手机号/姓名 → [REDACTED]）

### 6.3 法律免责
